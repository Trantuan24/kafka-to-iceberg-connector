TÓM TẮT NGUỒN THAM KHẢO CHO CUSTOM CDC VERSION CONTROL CONNECTOR

Mục tiêu: chọn các nguồn thật sự áp dụng được cho bài toán xây custom Kafka Connect Sink Connector ghi Iceberg, có xử lý max(version), INSERT/UPDATE/DELETE, dedup_key, Hive Catalog và S3/MinIO.

================================================================================
1. Apache Iceberg Kafka Connect official docs
Nguồn: https://iceberg.apache.org/docs/1.10.0/kafka-connect/

Tác dụng:
- Là tài liệu baseline chính thức cho Apache Iceberg Sink Connector trên Kafka Connect.
- Tham khảo cách connector official tổ chức config, message format, catalog config, Hive/REST/Glue/Nessie catalog, warehouse, S3/Hadoop configuration.
- Tham khảo các feature nền như commit coordination, exactly-once delivery semantics, multi-table fan-out, auto table creation và schema evolution.
- Dùng để đảm bảo custom connector của mình không lệch khỏi cách Iceberg/Kafka Connect vận hành chuẩn.

Áp dụng vào bài toán:
- Dùng để thiết kế phần config Kafka Connect.
- Dùng để tham khảo cách cấu hình Hive Catalog + warehouse + S3/MinIO.
- Dùng để so sánh giới hạn của connector official với logic custom của mình.
- Không dùng trực tiếp cho logic max(version), vì official connector không xử lý business version control theo format custom của mình.

================================================================================
2. Apache Iceberg Kafka Connect SMT / CDC Transforms docs
Nguồn: https://iceberg.apache.org/docs/1.10.0/kafka-connect/

Tác dụng:
- Tài liệu này có phần SMTs cho Iceberg Sink Connector như CopyValue, DmsTransform, DebeziumTransform, JsonToMapTransform, KafkaMetadataTransform.
- Cho thấy Iceberg official có hỗ trợ transform cho một số CDC format phổ biến như Debezium/AWS DMS.
- JsonToMapTransform hữu ích để hiểu cách xử lý JSON schema-less; KafkaMetadataTransform hữu ích để tham khảo cách inject topic/partition/offset.

Áp dụng vào bài toán:
- Dùng để chứng minh SMT chỉ phù hợp transform message, không phù hợp xử lý stateful max(version) + query Iceberg + I/U/D.
- Dùng để tham khảo cách add metadata Kafka nếu cần: topic, partition, offset.
- Không dùng trực tiếp vì message của mình là envelope custom: data[], key, type, version, ngay_cap_nhat.

3. databricks/iceberg-kafka-connect GitHub
Nguồn: https://github.com/databricks/iceberg-kafka-connect

Tác dụng:
- Đây là repo rất quan trọng vì là connector Iceberg Kafka Connect có hỗ trợ upsert mode, CDC field, id columns/primary key, schema evolution, multi-table routing.
- README có các config gần với bài toán như:
  + iceberg.tables.default-id-columns
  + iceberg.tables.cdc-field
  + iceberg.tables.upsert-mode-enabled
  + iceberg.tables.evolve-schema-enabled
- Là nguồn tham khảo tốt nhất cho cách thiết kế connector có upsert/CDC thay vì chỉ append-only.

Áp dụng vào bài toán:
- Tham khảo cách định nghĩa id-columns/primary key tương tự dedup_key của mình.
- Tham khảo cách bật upsert mode và CDC field tương tự type INSERT/UPDATE/DELETE.
- Tham khảo kiến trúc writer, commit, table routing, schema handling.
- Rất phù hợp để học pattern, nhưng không dùng nguyên xi vì format message và rule version control của mình khác.

4. DeepWiki - Upsert Mode của databricks/iceberg-kafka-connect
Nguồn: https://deepwiki.com/databricks/iceberg-kafka-connect/4.3-upsert-mode

Tác dụng:
- Giải thích dễ hiểu hơn về upsert mode trong databricks/iceberg-kafka-connect.
- Nêu rõ upsert mode hoạt động bằng cách thực hiện equality delete trước mỗi append, để thay thế record cũ có cùng primary key.
- Nêu yêu cầu quan trọng: target table phải là Iceberg format-version=2 và phải có identity fields/primary keys.

Áp dụng vào bài toán:
- Dùng trực tiếp cho logic UPDATE của mình:
  UPDATE = delete bản cũ theo dedup_key + append bản mới.
- Dùng cho logic DELETE:
  DELETE = row-level/equality delete theo dedup_key.
- Dùng để củng cố yêu cầu format-version=2 trong design.
- Dùng để tránh hiểu sai DeleteFiles là xóa row; thực tế cần row-level delete/equality delete hoặc file rewrite strategy.

5. getindata/kafka-connect-iceberg-sink GitHub
Nguồn: https://github.com/getindata/kafka-connect-iceberg-sink

Tác dụng:
- Repo không phải official và đã archive, nhưng rất sát với bài toán version control.
- Có các config gần ý tưởng của mình:
  + upsert=true
  + upsert.keep-deletes=true
  + upsert.dedup-column=__source_ts_ms
  + upsert.op-column=__op
- Cho thấy đã có tiền lệ connector Iceberg Sink tự xử lý upsert/dedup/op column.

Áp dụng vào bài toán:
- Tham khảo naming/config cho custom connector:
  + dedup column tương ứng version
  + op column tương ứng type
  + primary key/dedup key tương ứng dedup_key
- Tham khảo rule giữ record mới hơn theo timestamp/version.
- Tham khảo lựa chọn xử lý delete/tombstone.
- Không dùng nguyên xi vì repo archived và dựa trên assumptions khác, nhưng rất hữu ích để chứng minh hướng custom connector là khả thi.

Nguồn: https://github.com/rapid7/iceberg-kafka-connect

Tác dụng:

- Có giá trị như nguồn tham khảo bổ sung cho hướng connector có row mutation/upsert, ngoài databricks và official docs.

Áp dụng vào bài toán:
- Dùng để tham khảo thêm pattern về row mutation/update/delete trong connector Kafka Connect -> Iceberg.
- Dùng để đối chiếu thiết kế custom connector của mình với các implementation khác.
- Không phải nguồn chính, nhưng hữu ích nếu cần xem thêm cách đóng gói/build connector hoặc tổ chức runtime.

7. Apache Iceberg Java API docs
Nguồn: https://iceberg.apache.org/docs/latest/docs/api/

Tác dụng:
- Tài liệu chính thức về Iceberg Java API.
- Giải thích Table interface và các operation như:
  + newAppend
  + newDelete
  + newRewrite
  + newRowDelta
  + newTransaction
  + newScan
- Có phần transaction: cho phép gom nhiều operation và commit atomically bằng commitTransaction().

Áp dụng vào bài toán:
- Đây là nguồn bắt buộc nếu tự build custom connector dùng Iceberg API.
- Dùng cho phần load table, scan existing versions, append data files, row-level change, transaction.
- Đặc biệt quan trọng để sửa design: không nên commit INSERT/UPDATE/DELETE rời rạc nếu yêu cầu atomic batch; nên cân nhắc Iceberg transaction.
